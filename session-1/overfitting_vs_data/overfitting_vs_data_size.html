Overparameterized net on tiny vs. large data

A ~1.2k-parameter MLP (2→32→32→1, ReLU, Adam) is trained from scratch on a noisy two-moons binary classification task at three train sizes: 20, 200, 2000. Same fixed 2000-point held-out test set every time. Watch the train/test gap.

Claim: a high-capacity model on tiny data drives train loss to ~0 while held-out loss stays high; growing the dataset closes the gap. That's the model memorizing label noise it can't possibly generalize from at n=20, versus being forced to learn the actual signal once n=2000 makes memorization impossible. Data is everything.

Model: 2 → 32 (ReLU) → 32 (ReLU) → 1 (sigmoid), ~1,185 parameters, trained full-batch with Adam (lr=0.02), binary cross-entropy loss. Data: two interleaving moons with Gaussian coordinate noise (σ=0.25) and 12% random label flips (irreducible noise — no model gets test loss to 0). Same seeds every run for reproducibility. Everything runs client-side in plain JS, no ML libraries — full source is in this file.
