Emergent Clustering in Next-Token Embeddings

Claim: a model trained only to predict the next token in a tiny synthetic grammar will organize its embedding table by semantic category — even though category labels and similarity were never part of the training signal. The mechanism: same-category tokens are grammatically interchangeable, so they end up with near-identical next-token distributions, and gradient descent is forced to give them near-identical embeddings to produce that. Everything below runs live in this page — no server, no pretrained anything.

How the toy grammar works

Vocabulary (9 tokens), grouped into categories the model is never told about:

animals: cat, dog, cow fruits: apple, mango verbs: eat, chase, see boundary: <EOS>

Every sentence is generated as SUBJECT VERB OBJECT <EOS>, where SUBJECT is drawn uniformly from the 3 animals, VERB uniformly from the 3 verbs (independent of the subject), and OBJECT uniformly from all 5 animals+fruits (independent of the verb). This is what makes it a fair test: within each category, the members are 100% interchangeable — cat/dog/cow always play the same role with the same odds, so any two animals induce the exact same next-token statistics, and likewise for fruits and for verbs. The model only ever sees (current token → next token) pairs; it has no notion of "category".
